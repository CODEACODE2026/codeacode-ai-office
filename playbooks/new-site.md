# Playbook - New Site

## Quando utilizar
Use quando o usuario pedir site institucional, landing page, pagina de servico ou presenca web simples.

## Objetivo
Conduzir criacao de site com proposta clara, direcao visual premium e tecnologia proporcional.

## Sequencia de execucao
1. Entender oferta, publico e objetivo.
2. Consultar `project-library/landing-page.md` quando for pagina de conversao.
3. Consultar `inspiration/landing-pages.md`.
4. Definir UX: jornada, CTA e secoes.
5. Definir direcao criativa.
6. Validar visual com usuario.
7. Preparar tarefa pequena para Codex.
8. Validar responsividade e estados.
9. Preparar entrega ou deploy quando aprovado.

## Agentes envolvidos
JADE, Gerente de Projetos, Prompt Specialist, UX Lead, Diretor Criativo UI, DEV, UI Reviewer, QA e DevOps quando houver publicacao.

## Pontos de aprovacao
- aprovar objetivo e CTA;
- aprovar direcao visual;
- aprovar execucao tecnica;
- aprovar publicacao.

## Criterios de conclusao
- proposta clara;
- UX aprovada;
- direcao visual aprovada;
- site validado em mobile e desktop;
- proximos passos documentados.

## Erros comuns
- usar Next.js sem necessidade;
- copiar layout de referencia;
- criar hero generico;
- pular aprovacao visual;
- esquecer formulario ou WhatsApp.
