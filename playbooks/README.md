# Playbooks - AI Office Framework v2

## Objetivo
Guardar procedimentos operacionais reutilizaveis para conduzir projetos, correcoes e releases com menos improviso.

## Quando utilizar
Use `playbooks/` quando houver uma sequencia recorrente de trabalho.

## Como usar
1. Identificar o tipo de tarefa.
2. Ler apenas o playbook correspondente.
3. Consultar bibliotecas indicadas.
4. Confirmar pontos de aprovacao.
5. Acionar Codex somente com tarefa delimitada.

## Agentes envolvidos
- JADE;
- Gerente de Projetos;
- Prompt Specialist;
- Analista;
- Arquiteto;
- UX Lead;
- Diretor Criativo UI;
- DEV;
- UI Reviewer;
- QA;
- DevOps, quando houver deploy.

## Documentos iniciais
- `new-project.md`
- `new-site.md`
- `bugfix.md`
- `release.md`

## Regras
- Playbook nao substitui aprovacao do usuario.
- Playbook nao cria escopo sozinho.
- Playbook deve apontar bibliotecas, nao duplicar conhecimento.
- Codex so entra quando houver tarefa objetiva.
