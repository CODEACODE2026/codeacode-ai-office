# Playbook - New Project

## Quando utilizar
Use quando o usuario pedir novo projeto, sistema, SaaS, CRM, PDV, ERP, automacao ou plataforma.

## Objetivo
Conduzir o inicio de projeto com fluxo oficial, escopo claro e uso economico do Codex.

## Sequencia de execucao
1. JADE entende o pedido.
2. Gerente de Projetos classifica tipo e nivel.
3. Prompt Specialist prepara discovery e master prompt.
4. Analista define requisitos.
5. Arquiteto define arquitetura.
6. UX Lead define experiencia quando houver interface.
7. Diretor Criativo UI define direcao visual quando houver web.
8. Usuario aprova plano.
9. DEV/Codex executa apenas tarefa aprovada.
10. UI Reviewer, QA e DevOps entram conforme necessidade.

## Agentes envolvidos
JADE, Gerente de Projetos, Prompt Specialist, Analista, Arquiteto, UX Lead, Diretor Criativo UI, DEV, QA e DevOps.

## Documentos consultados
- `WORKFLOW.md`
- `DESIGN_PRINCIPLES.md`
- projeto em `projects/`

## Bibliotecas consultadas
- `project-library/`
- `knowledge/`
- `prompt-library/`
- `component-library/`
- `inspiration/`

## Pontos de aprovacao
- aprovar escopo inicial;
- aprovar direcao UX/UI quando houver web;
- aprovar cada tarefa para Codex;
- aprovar release/deploy.

## Criterios de conclusao
- tipo de projeto classificado;
- documentos iniciais criados;
- tarefas pequenas definidas;
- proxima acao aprovada.

## Erros comuns
- acionar Codex cedo demais;
- criar sistema inteiro em uma tarefa;
- pular UX/UI;
- nao consultar Project Library;
- nao pedir aprovacao.
