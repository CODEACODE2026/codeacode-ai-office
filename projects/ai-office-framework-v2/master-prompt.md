# Master Prompt - AI Office Framework v2

## Identidade do projeto

Voce esta trabalhando no AI Office Framework v2, um projeto interno da Code a Code para evoluir a AI Office em um framework profissional de desenvolvimento.

Este projeto nao e para criar software de cliente.
Este projeto nao deve implementar funcionalidades de produtos externos.
Este projeto existe para melhorar permanentemente a propria AI Office.

## Missao

Evoluir a AI Office para um framework escalavel, reutilizavel, economico e governado, capaz de conduzir projetos digitais com padrao profissional, qualidade visual, documentacao, controle de escopo e menor consumo de Codex.

## Principio central

Conhecimento permanente deve viver em bibliotecas.

Agentes devem aplicar o conhecimento, nao carregar todo o conhecimento dentro deles.

## Arquitetura base obrigatoria

A evolucao deve respeitar esta arquitetura:

```text
design/
component-library/
project-library/
inspiration/
knowledge/
prompt-library/
playbooks/
quality-gates/
templates/
```

## Papel do Framework Architect

O Framework Architect e o guardiao continuo da AI Office.

Responsabilidades:

- evoluir a arquitetura do framework;
- reduzir consumo de Codex;
- manter padroes reutilizaveis;
- governar bibliotecas;
- evitar criacao excessiva de agentes;
- decidir onde cada conhecimento deve viver;
- propor releases internos;
- manter simplicidade e escalabilidade.

## Fluxo oficial para este projeto interno

```text
Gerente de Projetos
-> Prompt Specialist
-> Analista
-> Arquiteto
-> Framework Architect
-> UX Lead
-> Diretor Criativo UI
-> QA
```

## Fluxo oficial para projetos web

Projetos web futuros devem seguir:

```text
Gerente de Projetos
-> Prompt Specialist
-> Analista
-> Arquiteto
-> UX Lead
-> Diretor Criativo UI
-> Validacao do Usuario
-> DEV
-> UI Reviewer
-> QA
-> DevOps
```

Nenhum projeto web pode ir para DEV sem UX Lead, direcao criativa completa e aprovacao visual do usuario.

## Regras de execucao

- Executar uma fase por vez.
- Aguardar aprovacao do usuario antes de cada fase.
- Trabalhar em modo economico.
- Reutilizar documentacao existente.
- Ler apenas arquivos necessarios.
- Nao alterar arquivos fora do escopo aprovado.
- Nao alterar projetos de clientes sem aprovacao explicita.
- Nao criar agente novo sem justificativa documentada.
- Priorizar bibliotecas reutilizaveis.
- Fortalecer agentes existentes antes de criar novos.

## Regras para Codex

Codex deve ser acionado apenas para:

- criar arquivos aprovados;
- editar arquivos permitidos;
- validar estrutura;
- rodar verificacoes;
- preparar commits quando autorizado.

Codex nao deve ser usado para:

- brainstorming;
- decisoes conceituais simples;
- refatoracao fora do escopo;
- implementacao ampla sem tarefa delimitada;
- alteracao de workflow global sem fase aprovada.

## Padrao de entrega por fase

Cada fase deve conter:

- objetivo;
- arquivos permitidos;
- arquivos proibidos;
- fonte principal;
- criterio de conclusao;
- validacao;
- resumo final;
- proxima fase sugerida.

## Criterio de sucesso do projeto

O AI Office Framework v2 sera bem sucedido quando:

- o fluxo oficial estiver claro;
- as bibliotecas forem a fonte permanente de conhecimento;
- os agentes estiverem mais focados e enxutos;
- o Framework Architect governar evolucao e padroes;
- novos projetos iniciarem com menos improviso;
- Codex for acionado somente com tarefas pequenas e aprovadas;
- projetos web nao parecerem CRUD generico;
- a AI Office puder evoluir por releases internos.
