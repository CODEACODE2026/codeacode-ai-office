# Background Assets

## Objetivo
Definir criterios para fundos visuais que sustentam conteudo sem prejudicar leitura.

## Quando utilizar
- Hero sections com texto sobre imagem.
- Bandas de pagina, CTAs e secoes de transicao.
- Texturas sutis, fotos contextuais ou padroes visuais de marca.

## Quando evitar
- Quando o fundo reduz contraste ou leitura.
- Quando for puramente decorativo e pesado.
- Quando parecer template generico.
- Quando competir com produto, mensagem ou CTA.

## Padroes de qualidade
- Contraste testado com texto real.
- Area segura para copy e botoes.
- Baixo ruido visual atras de informacao.
- Coerencia com paleta e direcao visual.

## Proporcao recomendada
- Hero desktop: 16:9 ou 21:9.
- Mobile: 9:16 ou 4:5.
- Bandas: 16:6 ou textura repetivel.

## Formatos
- WebP/AVIF para fotos e texturas.
- SVG/CSS para padroes simples.
- PNG quando transparencia for necessaria.

## Resolucao minima
- Hero: 1920px de largura.
- Background full HD: 1920x1080.
- Textura repetivel: 512px ou 1024px bem otimizado.

## Erros comuns
- Fundo escuro demais sem informacao util.
- Gradiente generico sem relacao com marca.
- Imagem borrada para esconder baixa qualidade.
- Peso alto impactando carregamento.

## Criterios de reutilizacao
- Reutilizar fundos como sistema de secoes, nao como enfeite isolado.
- Manter contraste consistente.
- Variar intensidade sem mudar linguagem visual.

## Palavras-chave recomendadas
- `subtle texture`, `office background`, `clean workspace`
- `abstract background minimal`, `brand pattern`, `soft gradient background`
- Combinar contexto + nivel de ruido + espaco para texto.
