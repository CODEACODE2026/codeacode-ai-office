# Mockup Assets

## Objetivo
Definir uso de mockups para apresentar produto digital, dashboard, app, site ou fluxo.

## Quando utilizar
- SaaS, ERP, CRM, dashboard, app ou plataforma.
- Hero com produto digital.
- Secoes de recursos que precisam mostrar interface.
- Prova visual de sistema real ou prototipo aprovado.

## Quando evitar
- Quando o produto ainda nao existe e o mockup prometer funcionalidade falsa.
- Quando a tela estiver ilegivel.
- Quando foto de contexto explicar melhor.
- Quando o mockup for generico demais.

## Padroes de qualidade
- Tela legivel nos pontos importantes.
- Interface coerente com o produto real.
- Moldura discreta, sem ocupar mais atencao que o conteudo.
- Dados sensiveis removidos ou anonimizados.

## Proporcao recomendada
- Desktop: 16:9 ou 16:10.
- Mobile: 9:16.
- Hero combinado: 3:2 ou 4:3.
- Cards: 4:3.

## Formatos
- PNG para mockups com transparencia.
- WebP para imagens finais otimizadas.
- Figma/source quando editavel for necessario.

## Resolucao minima
- Hero desktop: 1920px de largura.
- Card: 1000px de largura.
- Mobile mockup: 1080px de altura.

## Erros comuns
- Mockup bonito mas nao legivel.
- Dados falsos sem sinalizar que e demonstrativo.
- Perspectiva exagerada que atrapalha leitura.
- Repetir a mesma tela para todos os recursos.

## Criterios de reutilizacao
- Reutilizar moldura e angulo para consistencia.
- Atualizar screenshot quando a UI mudar.
- Usar telas diferentes para mensagens diferentes.

## Palavras-chave recomendadas
- `SaaS dashboard mockup`, `app screen mockup`, `laptop product mockup`
- `CRM interface mockup`, `ERP dashboard screenshot`, `analytics dashboard`
- Combinar produto + dispositivo + tela + estilo.
