# Icon Assets

## Objetivo
Definir criterios para icones em sites, landing pages e interfaces.

## Quando utilizar
- Acoes, beneficios, categorias, recursos e estados.
- Botoes, menus, listas de features e cards compactos.
- Apoio visual rapido sem competir com texto.

## Quando evitar
- Icones decorativos sem funcao.
- Icones que repetem obviedades do texto.
- Mistura de familias, espessuras e estilos.
- Icones complexos demais para tamanhos pequenos.

## Padroes de qualidade
- Mesmo estilo, peso, raio, preenchimento e grid visual.
- Significado reconhecivel.
- Contraste suficiente no fundo usado.
- Preferir bibliotecas consistentes quando o projeto ja usa uma.

## Proporcao recomendada
- Quadrado 1:1.
- UI comum: 16, 20 ou 24px.
- Cards/beneficios: 32 a 48px.
- Ilustrativo grande: ate 64px com cuidado.

## Formatos
- SVG para interface.
- Icon font somente se ja for padrao do projeto.
- PNG apenas para compatibilidade especifica.

## Resolucao minima
- SVG vetorial preferencial.
- PNG: exportar 2x ou 3x do tamanho exibido.

## Erros comuns
- Icone com significado ambiguo.
- Usar emoji como icone de UI profissional.
- Misturar outline e filled sem criterio.
- Icone desalinhado verticalmente no botao.

## Criterios de reutilizacao
- Reutilizar a mesma familia em todo o projeto.
- Criar mapa de icones para evitar duplicidade.
- Trocar icone se o contexto mudar de significado.

## Palavras-chave recomendadas
- `outline icons`, `linear icons`, `interface icons`
- `dashboard icons`, `business icons`, `service icons`
- Combinar acao/recurso + estilo + biblioteca quando existir.
