# Assets Library - AI Office Framework v2

## Objetivo
Orientar escolha, justificativa e uso de imagens e assets para sites, landing pages e interfaces publicas.

Esta biblioteca define regras e padrao de uso. Integracoes externas, como Freepik, ficam para uma fase futura.

## Quando usar
Use `assets-library/` quando o projeto precisar de:
- imagens para hero section;
- fotos de produto, servico, equipe, local ou contexto;
- referencias visuais para landing pages e sites;
- palavras-chave para pesquisa de imagens;
- orientacao de uso de assets para DEV;
- criterio para evitar imagens genericas.

## Quando nao usar
Nao use esta biblioteca para:
- baixar assets automaticamente;
- integrar bancos pagos ou APIs externas;
- substituir direcao criativa;
- definir identidade visual completa;
- escolher imagens sem contexto de publico, marca e objetivo.

## Principios
- Asset precisa servir ao objetivo da secao, nao apenas decorar.
- Imagem principal deve mostrar produto, servico, pessoa, lugar, resultado ou contexto real.
- Evitar imagens genericas, stock artificial, pessoas posando sem relacao com a oferta e fundos escuros sem informacao.
- Sites e landing pages devem usar assets coerentes com publico, marca, nivel de maturidade e promessa comercial.
- Toda imagem sugerida precisa ter justificativa de uso.
- Quando houver duvida entre imagem bonita e imagem clara, priorizar clareza.
- Nao usar asset protegido, marca de terceiros ou imagem de pessoa real sem permissao/licenca adequada.

## Padrao de recomendacao
Para cada asset sugerido, registrar:
- secao onde sera usado;
- objetivo do asset;
- tipo de asset: foto, mockup, screenshot, textura, icone, video curto ou ilustracao;
- descricao visual esperada;
- palavras-chave de pesquisa;
- criterio de escolha;
- riscos a evitar;
- orientacao para DEV sobre posicionamento, recorte, proporcao e responsividade.

## Palavras-chave
Palavras-chave devem combinar:
- segmento do cliente;
- publico-alvo;
- situacao de uso;
- emocao desejada;
- estilo visual;
- formato necessario.

Exemplo:
- `clinica odontologica recepcao moderna paciente sorrindo luz natural`
- `software dashboard financeiro notebook escritorio pequena empresa`
- `equipe tecnica instalacao energia solar telhado residencial`

## Uso por secao
Hero:
- priorizar imagem forte, especifica e imediatamente relacionada a oferta;
- evitar foto abstrata ou atmosferica demais;
- prever recorte desktop e mobile.

Prova social:
- usar logos, depoimentos, fotos reais ou evidencias quando disponiveis;
- evitar selos falsos ou numeros sem fonte.

Produto/servico:
- mostrar o produto, processo, resultado ou contexto real de uso;
- para SaaS, preferir screenshot/mockup legivel a imagem generica de tecnologia.

CTA:
- usar asset apenas se reforcar acao ou confianca;
- nao competir visualmente com o botao principal.

## Regra de economia
Sugerir poucos assets bons. Nao criar catalogos extensos sem necessidade.

## Relacao com outros agentes
- UX Lead define estrutura e necessidade das secoes.
- Conversion Strategist define promessa e prioridade comercial.
- Brand Strategist define percepcao e tom de marca.
- Asset Manager escolhe e justifica assets coerentes.
- Diretor Criativo UI integra os assets na direcao visual.
- DEV implementa usando as orientacoes aprovadas.
