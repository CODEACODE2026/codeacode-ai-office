# Landing Page

## Objetivo
Landing page e uma pagina focada em conversao: capturar lead, vender uma oferta, apresentar servico ou direcionar o visitante para uma acao principal.

## Funcionalidades esperadas
- hero com oferta clara;
- chamada principal para acao;
- beneficios;
- prova social;
- detalhes do servico/produto;
- perguntas frequentes;
- formulario ou link de contato;
- tracking basico quando aprovado.

## Modulos obrigatorios
- Hero com proposta de valor.
- Secao de beneficios.
- Secao de prova ou autoridade.
- CTA principal.
- Formulario ou contato.
- Rodape simples.

## Modulos opcionais
- Depoimentos.
- Comparativo.
- Galeria.
- FAQ.
- Mapa.
- Integracao WhatsApp.
- Captura de lead com CRM.

## Integracoes comuns
- WhatsApp;
- formulario por email;
- CRM;
- Google Analytics;
- Meta Pixel;
- Tag Manager.

## Requisitos nao funcionais
- carregamento rapido;
- mobile first;
- SEO basico;
- acessibilidade minima;
- copy clara;
- imagens otimizadas;
- compatibilidade com hospedagem escolhida.

## Checklist minimo
- oferta entendida em poucos segundos;
- CTA visivel;
- formulario testado;
- responsividade validada;
- links funcionando;
- imagens leves;
- titulo e meta description definidos.

## Erros comuns
- visual generico;
- excesso de texto;
- CTA fraco;
- hero sem oferta;
- formulario longo;
- site pesado;
- usar stack complexa sem necessidade.

## Boas praticas
- usar linguagem direta;
- priorizar uma acao principal;
- mostrar prova de confianca;
- evitar distracoes;
- usar imagem real ou relevante;
- adaptar tecnologia ao nivel do projeto.

## Reutilizacao pelos agentes
Gerente usa para classificar escopo. Prompt Specialist usa para discovery. UX Lead usa para jornada de conversao. Diretor Criativo UI usa para direcao visual. DEV usa apenas apos aprovacao do plano.
