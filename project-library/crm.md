# CRM

## Objetivo
CRM organiza relacionamento com clientes, oportunidades comerciais, tarefas e funil de vendas.

## Funcionalidades esperadas
- cadastro de leads e clientes;
- funil de oportunidades;
- historico de interacoes;
- tarefas e lembretes;
- filtros e segmentacao;
- status de negociacao;
- dashboards comerciais;
- permissoes por usuario ou equipe.

## Modulos obrigatorios
- Autenticacao.
- Leads/clientes.
- Oportunidades.
- Funil.
- Atividades.
- Usuarios/permissoes.
- Dashboard basico.

## Modulos opcionais
- Importacao de contatos.
- Integracao WhatsApp.
- Email.
- Automacoes.
- Campos personalizados.
- Propostas.
- Relatorios avancados.
- Multiempresa.

## Integracoes comuns
- WhatsApp;
- email;
- formulario de site;
- planilhas;
- telefonia;
- calendario;
- ferramentas de marketing.

## Requisitos nao funcionais
- filtros rapidos;
- historico confiavel;
- permissao por perfil;
- interface para uso diario;
- boa busca;
- responsividade;
- baixa complexidade no cadastro.

## Checklist minimo
- criar lead;
- mover oportunidade no funil;
- registrar atividade;
- filtrar por status;
- atribuir responsavel;
- consultar historico;
- visualizar indicadores basicos.

## Erros comuns
- cadastro pesado demais;
- funil rigido demais;
- pouca visibilidade de proximas acoes;
- dashboard decorativo;
- falta de historico;
- permissoes esquecidas.

## Boas praticas
- priorizar proxima acao;
- reduzir campos obrigatorios;
- facilitar busca;
- destacar oportunidades paradas;
- manter funil simples;
- medir conversao por etapa.

## Reutilizacao pelos agentes
Gerente usa para escopo comercial. Analista usa para requisitos de funil. UX Lead usa para jornada de vendedor. UI Reviewer verifica densidade, filtros e estados.
