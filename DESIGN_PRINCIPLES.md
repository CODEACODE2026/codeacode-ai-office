# DESIGN PRINCIPLES - Code a Code

A Code a Code não entrega sistemas com aparência genérica.

Todo projeto web deve parecer moderno, profissional e confiável.

## Padrão visual obrigatório

- Aparência SaaS premium
- Boa hierarquia visual
- Espaçamento consistente
- Tipografia clara
- Componentes reutilizáveis
- Responsividade real
- Desktop, tablet e mobile bem cuidados
- Ícones modernos
- Estados visuais bem definidos

## Todo sistema deve conter

- Loading
- Skeleton
- Empty state
- Error state
- Success state
- Toasts
- Modais
- Confirmações
- Hover
- Focus
- Microinterações
- Animações leves
- Feedback visual em ações importantes

## Referências permitidas

- Stripe
- Linear
- Vercel
- Notion
- Clerk
- Shopify
- Tailwind UI
- shadcn/ui
- Magic UI
- Aceternity UI
- Origin UI

## Regra principal

Nenhum projeto deve parecer CRUD genérico.
