# API Testing

## Objetivo
Validar endpoints e contratos principais.

## Validar
- healthcheck
- autenticacao quando houver
- endpoints principais
- payload valido e invalido
- status codes
- mensagens de erro

## Evidencias
- comandos curl, HTTP client ou teste automatizado
- resposta resumida
- ambiente usado

## Reprovacao
Endpoint principal sem teste real ou contrato divergente do frontend.
