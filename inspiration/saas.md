# SaaS Inspiration

## Objetivo
Orientar interfaces SaaS modernas, confiaveis e escalaveis.

## Caracteristicas visuais
- navegacao clara;
- dashboard util;
- componentes consistentes;
- estados bem definidos;
- visual premium sem excesso;
- informacao organizada por prioridade.

## Padroes modernos recomendados
- sidebar ou topbar previsivel;
- onboarding simples;
- empty states orientativos;
- modais para decisoes curtas;
- tabelas com filtros;
- settings organizados;
- feedback visual em toda acao.

## Referencias de mercado
- Linear: produtividade, foco e densidade.
- Stripe: confianca, dados financeiros e clareza.
- Vercel: experiencia tecnica e simplicidade.
- Notion: modularidade e leveza.
- Shopify: gestao operacional amigavel.
- shadcn/ui e Tailwind UI: componentes consistentes.

Usar como referencia de padrao profissional, nao como copia.

## Principios de layout
- navegacao sempre compreensivel;
- conteudo principal com largura controlada;
- areas densas com hierarquia clara;
- settings separados de operacao diaria;
- estados vazios devem guiar o usuario.

## Principios de UX
- reduzir tempo ate valor;
- evitar telas sem proxima acao;
- proteger acoes destrutivas;
- manter consistencia entre modulos;
- permitir retorno facil ao fluxo principal.

## Principios de UI
- componentes reutilizaveis;
- cores funcionais;
- tipografia limpa;
- iconografia consistente;
- foco e hover claros;
- responsividade real.

## Animacoes e microinteracoes
- skeletons;
- toasts;
- hover em acoes;
- transicoes de modal;
- feedback de salvamento;
- animacoes leves de onboarding.

## Erros comuns
- parecer admin template;
- excesso de cards;
- modais demais;
- settings confuso;
- estados vazios abandonados;
- dashboard sem acao.

## Reutilizacao pelos agentes
Gerente usa para classificar SaaS. UX Lead define jornadas. Diretor Criativo UI define sistema visual. DEV reutiliza componentes. UI Reviewer barra visual generico.
