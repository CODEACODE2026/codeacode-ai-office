# Inspiration Library - AI Office Framework v2

## Objetivo
Guardar referencias visuais e boas praticas de mercado para orientar direcao criativa sem copiar layouts ou conteudos protegidos.

## Quando usar
Use `inspiration/` antes de definir:
- conceito visual;
- direcao criativa;
- layout de landing page;
- dashboard;
- login;
- SaaS;
- padrao de microinteracoes.

## Quando evitar
Nao use esta biblioteca para:
- copiar interface existente;
- substituir UX Lead;
- substituir direcao criativa;
- escolher stack tecnica;
- justificar visual generico.

## Referencias de mercado
Referencias como Stripe, Linear, Vercel, Notion, Clerk, Shopify, Tailwind UI, shadcn/ui, Magic UI, Aceternity UI e Origin UI devem ser usadas apenas como inspiracao de boas praticas.

## Regras
- Inspirar, nao copiar.
- Adaptar ao contexto do projeto.
- Priorizar clareza e confianca.
- Evitar CRUD generico.
- Considerar desktop, tablet e mobile.
- Incluir loading, empty, error, success, hover, focus e microinteracoes.
