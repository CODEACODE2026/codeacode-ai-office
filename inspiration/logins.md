# Logins Inspiration

## Objetivo
Orientar telas de login profissionais, seguras e objetivas.

## Caracteristicas visuais
- foco no acesso;
- formulario claro;
- identidade visual discreta;
- erro legivel;
- recuperacao de senha visivel quando houver;
- layout responsivo.

## Padroes modernos recomendados
- formulario compacto;
- campos com labels visiveis;
- botao principal evidente;
- feedback de erro especifico;
- loading ao entrar;
- protecao contra estados ambíguos.

## Referencias de mercado
- Clerk: clareza em autenticacao.
- Notion: simplicidade e foco.
- Vercel: login tecnico limpo.
- Linear: visual sobrio e profissional.
- Shopify: confianca para operacao comercial.

Usar como referencia de confianca e clareza, nao como copia.

## Principios de layout
- formulario central ou painel simples;
- logo sem competir com campos;
- espaco suficiente em mobile;
- mensagens perto da acao;
- evitar layout de landing quando for sistema.

## Principios de UX
- usuario entende erro e proximo passo;
- enter deve enviar quando seguro;
- senha deve poder ser revelada;
- loading evita duplo envio;
- acesso bloqueado deve ser explicado.

## Principios de UI
- contraste alto nos campos;
- foco visivel;
- botao com estado loading;
- erro com texto e cor;
- tipografia legivel.

## Animacoes e microinteracoes
- feedback ao focar campo;
- loading no botao;
- transicao leve de erro;
- evitar animacao decorativa intensa.

## Erros comuns
- erro generico demais;
- campo sem label;
- botao pequeno;
- visual de marketing em tela operacional;
- falta de feedback no envio;
- layout quebrado em mobile.

## Reutilizacao pelos agentes
UX Lead define fluxo. Diretor Criativo UI define tom visual. DEV implementa estados. QA valida erro, loading, foco e responsividade.
