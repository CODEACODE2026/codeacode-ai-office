# Dashboards Inspiration

## Objetivo
Orientar dashboards claros, densos e uteis para decisao.

## Caracteristicas visuais
- indicadores bem hierarquizados;
- periodo visivel;
- alertas destacados;
- grid consistente;
- cards compactos;
- graficos simples;
- caminhos para detalhe.

## Padroes modernos recomendados
- resumo no topo;
- alertas acionaveis;
- filtros proximos ao contexto;
- tabelas com overflow controlado;
- graficos com legenda clara;
- estados vazios bem tratados.

## Referencias de mercado
- Linear: densidade, clareza e foco operacional.
- Stripe Dashboard: indicadores financeiros e confianca.
- Shopify Admin: gestao comercial com hierarquia clara.
- Notion: composicao limpa e modular.
- Tailwind UI: padroes consistentes de painel.

Usar como referencia de organizacao, nao como copia.

## Principios de layout
- indicador importante vem primeiro;
- grid deve facilitar comparacao;
- detalhamento fica abaixo do resumo;
- filtros nao devem dominar a tela;
- evitar card decorativo.

## Principios de UX
- usuario deve saber onde agir;
- dados vazios precisam explicar o motivo;
- alertas devem ter proxima acao;
- comparar periodos deve ser simples;
- densidade deve seguir frequencia de uso.

## Principios de UI
- numeros legiveis;
- cores por funcao;
- contraste adequado;
- iconografia discreta;
- tabelas responsivas;
- badges para status.

## Animacoes e microinteracoes
- skeleton em carregamento;
- hover em linhas clicaveis;
- destaque em alerta novo;
- transicao leve em filtros;
- feedback ao atualizar dados.

## Erros comuns
- dashboard decorativo;
- graficos sem decisao;
- periodo oculto;
- cards grandes demais;
- cores em excesso;
- mobile inutilizavel.

## Reutilizacao pelos agentes
Analista define indicadores. UX Lead prioriza decisoes. Diretor Criativo UI define composicao. UI Reviewer valida densidade. QA valida estados.
