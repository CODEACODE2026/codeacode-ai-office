# Component Library - AI Office Framework v2

## Objetivo
Guardar padroes reutilizaveis de componentes para interfaces da Code a Code.

Esta biblioteca deve ajudar UX Lead, Diretor Criativo UI, DEV, UI Reviewer e QA a manter consistencia visual e evitar interfaces genericas.

## Quando usar
Use `component-library/` ao definir ou revisar:
- telas web;
- dashboards;
- formularios;
- botoes;
- cards;
- fluxos operacionais;
- estados de interface.

## Quando nao usar
Nao use esta biblioteca para:
- decidir regra de negocio;
- substituir direcao criativa;
- definir stack tecnica;
- criar layout sem entender a jornada;
- copiar componente sem adaptar ao contexto.

## Arquivos iniciais
- `buttons.md`
- `forms.md`
- `cards.md`
- `dashboard.md`

## Regra de reutilizacao
Cada componente deve ser aplicado com criterio. Um componente reutilizavel nao e um bloco visual fixo, e um padrao de decisao.

## Regras gerais
- Todo componente deve ter estado default, hover, focus, disabled e loading quando aplicavel.
- Componentes devem ser responsivos.
- Acoes criticas precisam de confirmacao.
- Textos devem caber no componente.
- Interface operacional deve priorizar clareza e velocidade.
